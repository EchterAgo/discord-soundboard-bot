Latency Metrics Explained
  • E2E (End-to-End): Complete latency from user click to first audio byte (includes network + server)
  • C→S (Client to Server): Network latency from click to server receiving request
  • Server: Server-side processing (queue + processing + decode)
  • Queue: Time from server receive to being added to queue
  • Proc (Processing): Time from queue to FFmpeg stream start
  • Decode: Time from FFmpeg start to first byte output
  • Duration: Total playback time
  • Latency Badges: < 100ms 100-300ms > 300ms
